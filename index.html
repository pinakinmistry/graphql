Problems with REST APIs? $P

  • Hard coded; can't adopt with changing clients' needs
  • Not built as per clients' need/intent
  • Under/over Fetching $P
  • Requests Responses Waterfall $P
  • API versioning
  • Breaking Changes
  • API Documentation
  • Maintenance Nightmare
  • No Componentization
  • Getting older

Can we do better?

  • Efficient data loading
  • Client should be in control
  • No API versioning
  • A consistent way of defining and fetching data
  • Independent of framework/tech stack/platform
  • Auto documentation as we develop
  • Fastest development for rapid feature releases

What is GraphQL?

  • Query Language for API
  • Request decides Response
  • Single API endpoint
  • Enables declarative data definition and fetching
  • Independent of where data is coming from
  • Demand Driven API
  • New API standards/specs
